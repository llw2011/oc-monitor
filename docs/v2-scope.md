# OC-Monitor V2 Scope（体验与权限阶段）

> 目标：在 V1 可用基线上，把“能用”升级为“好用、可控、可扩展”。

---

## 1) V2 必做目标

1. **监控台 UI 升级**（Mission Control 风格）
2. **信息分层可见**（默认脱敏，授权后完整）
3. **Token 解锁机制**（便于临时完整查看）
4. **管理员登录与会话鉴权**
5. **基础主题切换（亮/暗）**

---

## 2) 非目标（V2 不做）

- 多租户与组织级权限模型（RBAC 细粒度）
- 外部告警通知（Telegram/Webhook）
- 复杂报表系统与历史分析平台
- 高可用集群与分布式存储

---

## 3) 前端改造范围

### 3.1 视觉与交互
- 深色卡片 + KPI 概览
- Tab 化信息组织（节点、日志、告警、健康）
- 节点卡状态色（online/offline/stale）
- Toast 反馈（成功/失败/警告）

### 3.2 可用性
- 首屏信息密度优化
- 状态标签统一命名
- 关键动作二次确认（高风险操作）

---

## 4) 鉴权与可见性策略

### 4.1 未授权视图（默认）
- 允许查看总体状态
- 对敏感字段做掩码/降精度显示

### 4.2 授权后视图
- 完整节点详情
- 高级操作入口（例如告警处理）

### 4.3 会话安全
- 登录后颁发会话
- 支持登出与会话有效期控制

---

## 5) API 扩展边界（V2）

- `POST /api/admin/login`
- `POST /api/admin/logout`
- `GET /api/admin/me`
- Token 解锁相关接口（按实现保留）

---

## 6) 验收标准

- [x] UI 升级后首屏可读性明显提升
- [x] 默认脱敏 + 授权完整策略可用
- [x] 登录/登出闭环可用
- [x] 主题切换可用

---

## 7) 对 V2.5 的输入

V2 完成后，进入 V2.5 稳态化阶段，重点补齐：
- 告警中心
- 审计导出
- 数据留存策略
- 健康总览
- ACK / 静默机制
